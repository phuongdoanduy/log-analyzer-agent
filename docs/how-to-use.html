Method 1

REST API

Step 1: Create Session

# Get access token TOKEN=$(gcloud auth print-access-token) # Create session curl -X POST \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ "https://europe-west6-aiplatform.googleapis.com /v1/projects/klara-prod/locations/europe-west6 /reasoningEngines/RESOURCE_ID:query" \ -d '{ "class_method": "async_create_session", "input": {"user_id": "u_123"} }'

Step 2: Send Request

# Send analysis request (SSE streaming) curl -X POST \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ "https://europe-west6-aiplatform.googleapis.com /v1/projects/klara-prod/locations/europe-west6 /reasoningEngines/RESOURCE_ID:streamQuery?alt=sse" \ -d '{ "class_method": "async_stream_query", "input": { "user_id": "u_123", "session_id": "SESSION_ID", "message": "Analyze errors in dev-vn last 2h" } }'
✓ Direct control ✓ CI/CD friendly △ Verbose
Method 2

Python SDK

import vertexai from vertexai import agent_engines # 1. Connect to deployed agent vertexai.init(project="klara-prod", location="europe-west6") agent = agent_engines.get("projects/klara-prod/locations/europe-west6/reasoningEngines/RESOURCE_ID") # 2. Create session session = agent.create_session(user_id="u_123") # 3. Send analysis request for event in agent.stream_query( user_id="u_123", session_id=session["id"], message="Analyze errors in dev-vn last 2 hours" ): print(event)
✓ Clean API ✓ Type safe ✓ App integration

Method 4

CLI Wrapper Script

#!/bin/bash # analyze-logs.sh — Call deployed log-analyzer-agent RESOURCE_ID="${LOG_ANALYZER_RESOURCE_ID}" PROJECT="klara-prod" REGION="europe-west6" TOKEN=$(gcloud auth print-access-token) # 1. Create session SESSION=$(curl -s -X POST \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ "https://${REGION}-aiplatform.googleapis.com/v1/projects/${PROJECT} /locations/${REGION}/reasoningEngines/${RESOURCE_ID}:query" \ -d '{"class_method":"async_create_session","input":{"user_id":"cli"}}') SESSION_ID=$(echo $SESSION | jq -r '.output.id') # 2. Send analysis request curl -s -X POST \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ "https://${REGION}-aiplatform.googleapis.com/v1/projects/${PROJECT} /locations/${REGION}/reasoningEngines/${RESOURCE_ID}:streamQuery?alt=sse" \ -d "{ \"class_method\":\"async_stream_query\", \"input\":{\"user_id\":\"cli\",\"session_id\":\"${SESSION_ID}\",\"message\":\"$1\"} }"
# Usage: ./analyze-logs.sh "Analyze errors in dev-vn last 2 hours" ./analyze-logs.sh "What hit prod last hour?" ./analyze-logs.sh "Check luz-baumer logs in dev-vn"
